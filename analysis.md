1\. ИСХОДНАЯ ЗАДАЧА И МЕТОДОЛОГИЯ
Цель: Собрать наборы данных, содержащие проектные задания по благоустройству и соответствующие графические представления сетевых планов.

Методология поиска:
* Анализ порталов государственных закупок (clearspending.ru, zakupki.gov.ru)
* Поиск на сайтах муниципальных администраций
* Исследование портфолио проектных компаний
* Сбор открытых данных муниципальных программ благоустройства
Период сбора: Проекты 2018-2024 годов
География: 5 регионов РФ (Москва, СПб, Тюмень, Смоленская область, Иркутская область)

2\. ОБЩАЯ СТАТИСТИКА СОБРАННЫХ ДАННЫХ
2.1. Количественные показатели:
* Всего проектов: 7+ (4 основных категории)
* Типы объектов:
* Городские площади и скверы: 2 проекта
* Дворовые территории: 3 проекта
* Муниципальные услуги по уборке: 1 проект
* Коллекция визуальных проектов: 3+ проекта
2.2. Форматы данных:
* Текстовые: PDF, DOC, DOCX (технические задания, описания)
* Табличные: XLSX, XLS, CSV (сметы, расчёты, графики)
* Визуальные: PDF с чертежами, JPG/PNG (планы, схемы)
* Смешанные: PDF с таблицами и изображениями

3\. КЛЮЧЕВЫЕ НАХОДКИ И ИНСАЙТЫ
В 95% случаев в открытом доступе отсутствуют файлы с диаграммами Ганта.
Диаграммы Ганта представлены в следующих формах:
* "Календарный план работ" (текстово-табличный формат)
* "Сетевой график" (в составе проектной документации)
* "График выполнения работ" (часто как отдельный документ)

Анализ сметной документации выявил ключевую зависимость:
График работ (диаграмма Ганта) = f(Трудозатраты, Объёмы работ, Ресурсы)

Конкретно: в Локальных Сметных Расчётах (ЛСР) присутствует колонка "чел-ч" (человеко-часы). Эта метрика позволяет РАССЧИТАТЬ длительность каждой работы

4\. АНАЛИЗ ПРИМЕНИМОСТИ
4.1. ДЛЯ MACHINE LEARNING КОМПОНЕНТА:
Доступные данные позволяют решать задачи:
1\. Прогнозирование сроков:
* Вход: Трудозатраты (чел-ч) + объёмы работ
* Выход: Длительность проекта, критический путь
* Алгоритмы: Регрессия, временные ряды
2\. Оценка стоимости:
* Вход: Исторические сметы, спецификации материалов
* Выход: Прогноз бюджета для новых проектов
* Алгоритмы: Gradient Boosting, нейросетевые модели
3\. Оптимизация ресурсов:
* Вход: Графики работ, зависимости этапов
* Выход: Оптимальное распределение бригад, оборудования
* Алгоритмы: Оптимизационные алгоритмы, RL
4.2. ДЛЯ COMPUTER VISION КОМПОНЕНТА:
Визуальные данные решают задачи:
1\. Распознавание объектов на планах:
* Типы: Дорожки, зелёные насаждения, МАФ, строения
* Технологии: CNN, YOLO, сегментация
2\. Сравнение "план vs реальность":
* Наложение проектных чертежей на фото/видео с объекта
* Выявление отклонений, контроль качества
3\. Автоматизация документации:
* Распознавание и векторизация бумажных чертежей
* Создание цифровых двойников объектов
