# АНАЛИЗ ПРОЕКТА: Уборка и благоустройство, Санкт-Петербург

## 1. ХАРАКТЕРИСТИКА ПРОЕКТА
**Тип:** Муниципальный контракт на оказание услуг
**Объект:** Городская территория (улицы, дворы, общественные пространства)
**Специфика:** Регулярные работы, а не разовый проект строительства

## 2. ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)
1. **Место, условия и сроки:** Чёткие временные рамки и географические зоны
2. **Общие требования:** Стандарты качества (частота уборки, нормы содержания)
3. **Состав услуг:** Детализированный перечень регулярных работ

**Применение в ML:**
- **Прогнозирование нагрузки:** На основе сезонности, погоды, событий
- **Оптимизация маршрутов:** Для мобильных бригад
- **Контроль качества:** Сравнение требований ТЗ с фактическим состоянием (по фото/видео)

## 3. РАСЧЁТ НМЦК (Начальная Максимальная Цена Контракта)
- Формула расчёта: `НМЦК = ∑(Объём_работы × Цена_за_единицу)`
- Детализация по видам работ (уборка, вывоз, обработка)
- Сезонные коэффициенты (зима/лето)

**Ценность для ML-системы:**
1. **Структурированные финансовые данные:** Прямая связь "работа → стоимость"
2. **Базис для прогнозирования:** Исторические цены для будущих контрактов
3. **Валидация данных:** Возможность проверки реальных затрат против плановых

## 4. ВЫВОДЫ

### **ПРИМЕНИМОСТЬ ДАННЫХ:**
1. **Обучение моделей сезонности:** Пики нагрузки весной (таяние снега) и осенью (листопад)
2. **Оптимизация ресурсов:** Распределение бригад по районам на основе данных НМЦК
3. **Контроль исполнения:** Сравнение плановых частот уборки с фактическими данными с камер

### **ПРОБЛЕМЫ И ОГРАНИЧЕНИЯ:**
- **Нет визуальных данных:** Отсутствуют эскизы, планы территорий
- **Высокая динамика:** Погодные факторы сильно влияют на выполнение
- **Субъективность оценки:** "Чистота" — относительный показатель
